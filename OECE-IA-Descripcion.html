Resumen Ejecutivo

OECE-IA es un asistente virtual especializado en el sistema de contrataciones públicas del Perú, desarrollado para apoyar a funcionarios, servidores públicos y proveedores del Estado en la interpretación y aplicación de la normativa vigente.

El sistema combina un modelo de lenguaje local (sin dependencia de APIs externas de pago) con una base de conocimiento vectorial alimentada por los documentos oficiales de la OECE: leyes, reglamentos, directivas, resoluciones y pronunciamientos institucionales.

Funcionalidades Principales
  • Chat con IA especializada: respuestas basadas en la normativa vigente de contrataciones del Estado peruano.
  • RAG (Retrieval-Augmented Generation): el sistema recupera fragmentos relevantes de 362+ documentos antes de generar cada respuesta.
  • Doble base de conocimiento: colección pública OECE + documentos privados por usuario filtrados por caso/expediente.
  • Gestión de casos: los usuarios organizan conversaciones por expediente o proyecto de contratación.
  • Subida de documentos: usuarios y administradores pueden cargar PDFs, DOCX y TXT al sistema.
  • Panel de administración: gestión de la base de conocimiento, administración de usuarios y asignación de roles.
  • Historial persistente: todas las conversaciones se guardan en Firestore y se recuperan entre sesiones.
  • Autenticación segura: Google Sign-In con Firebase Auth. Control de roles admin/usuario.
  • Interfaz responsiva: layout adaptado para escritorio (panel lateral de casos) y móvil.
  • Sugerencias de preguntas: preguntas frecuentes preconfiguradas para orientar al usuario.

Flujo de una Consulta
1
El usuario escribe su consulta
Desde la app web o móvil, dentro de un caso/expediente activo o en modo general.
2
Búsqueda semántica en ChromaDB
El backend convierte la consulta en un vector y busca los 3 fragmentos más relevantes de la base OECE y los documentos del usuario.
3
Construcción del prompt aumentado
Se combina el contexto recuperado + los últimos 6 mensajes del historial + la consulta del usuario en un prompt estructurado.
4
Generación de respuesta con Ollama
El LLM local genera la respuesta citando las fuentes relevantes. El sistema prompt garantiza tono formal e institucional.
5
Respuesta mostrada y guardada
La respuesta aparece en el chat con las fuentes documentales citadas. Se guarda automáticamente en Firestore.

Gestión Documental
Formato Procesamiento Uso
PDF pdfplumber — extracción página a página con número de página Leyes, reglamentos, directivas, resoluciones
DOCX python-docx — párrafo a párrafo Bases, TDR, expedientes técnicos
TXT / MD Lectura directa UTF-8 Notas, minutas, guías internas

Chunking inteligente: los documentos se dividen en fragmentos de 800 caracteres con solapamiento de 150 caracteres, respetando límites de párrafo y oración para preservar el contexto semántico.

Deduplicación automática: el sistema detecta archivos ya indexados mediante hash MD5, evitando duplicados al actualizar documentos.

7
Seguridad y Privacidad
  • LLM completamente local: ninguna consulta sale del servidor hacia APIs externas de terceros.
  • Autenticación Firebase: solo usuarios con cuenta Google autorizada pueden acceder al sistema.
  • Control de roles: administradores y usuarios con permisos diferenciados sobre la base de conocimiento.
  • Comunicación cifrada: TLS/HTTPS en todos los endpoints mediante Traefik con certificados automáticos.
  • Aislamiento de red: los servicios internos (Ollama, ChromaDB) no son accesibles desde internet.
  • Documentos por usuario: cada usuario solo accede a sus propios documentos privados.

Pendiente de hardening: externalizar configuraciones sensibles (emails admin, Client ID de Google) a variables de entorno antes de despliegue en producción ampliado.
